domStyle(styles) → {function}

Agrega los estilos especificados en el objeto de estilos. Cada clave equivale a los estilos usados en DOM, por ejemplo, `color`, `backgroundColor`, `justifyContent`. Observa que son las mismas propiedades que en CSS, sólo que al estilo datasets, usando la notación cammelCase en lugar de guiones `-`. Los mutadores pueden ser aplicados mediante mutate o individualmente.
Parameters:
Name Type Description
styles Object.<string, string> Objeto de estilos, cada clave equivale a element.style.keyName

Muta los estilos de un botón

const button = inlineHTML(`<button>hello world</button>`, "button");

Muta los estilos de una imagen

const image = inlineHTML(`<img src="http://placekitten.com/400">`, "img");

domStyle({ border: "10px solid white", boxShadow: "0px 0px 4px 4px rgba(0, 0, 0, 0.2)" })(image);

Reutiliza un mutador

const imageMutator = domStyle({ border: "10px solid white", boxShadow: "0px 0px 4px 4px rgba(0, 0, 0, 0.2)" });
const image1 = inlineHTML(`<img src="http://placekitten.com/400">`, "img");
const image2 = inlineHTML(`<img src="http://placekitten.com/500">`, "img");
// Observa que el mutador se puede aplicar individualemente
imageMutator(image1);
imageMutator(image2);

inlineHTML(html, query) → {HTMLElement}

Crea un elemento DOM a partir de un fragmento HTML. Estos serán contenidos en un div por lo que no es necesario definir un div principal. Si se define un query, en lugar de devolver el div principal, buscará mediante query selector el elemento específico y lo devolverá. De este modo se podrá usar una parte específica del fragmento HTML.
Parameters:
Name Type Default Description
html string Fragmento HTML que representa la interfaz. Serán contenidos en un div
query string null Query opcional para devolver un elemento específico usando `querySelector`

Examples

Crear un elemento DOM a partir de un fragmento HTML

const button = inlineHTML(`<button>hello</button>`);
// button es `<div><button>hello</button></div>`

Crear un elemento DOM a partir de un fragmento HTML usando un query

const button = inlineHTML(`<button>hello</button>`, "button");
// button es `<button>hello</button>`

Colocar una imagen el el body

const image = inlineHTML(`<img src="http://placehold.it/400">`, "img");
document.body.appendChild(image);

(async) load(name, baseURL, ext)

Genera una etiqueta script que carga el archivo desde la url. La url base por defecto es relativa al archivo HTML que llama a los componentes, sin embargo, se puede mandar a llamar cualquier CDN que almacene el script. Si la url ya ha sido consultada previamente será ignorada.

ext string .js Extensión del archivo

Aplicador Muta un elemento cualquiera mediante mutadores. La función encapsula un elemento y devuelve una segunda función que recibe en sus parámetros mutadores (funciones de mutación) y se las aplica al elemento encapsulado.

Examples

Sintaxis

// mutate(element)(
//   mutator_1(...config_1),
//   mutator_2(...config_2),
//   ...,
//   mutator_N(...config_N),
//);
mutate("button")(
 domAttribute({ innerText: "click here!" }),
 domStyle({ color: "blue" }),
 domEventCallback("click", () => alert("hi")),
);

Crear un aplicador a partir de un string sin mutadores

const button = mutate("button")();
document.body.appendChild(button);

Crear un aplicador a partir de un string con mutadores

const button = mutate("button")(
 domStyle({ color: "cornflowerblue" }),
 domAttribute({ innerText: "Hello world" }),
);
document.body.appendChild(button);

Crear una función que devuelva un aplicador configurado

const Button = (text, color) => mutate("button")(
 domStyle({ color }),
 domAttribute({ innerText: text }),
);
const button = Button("hello world", "crimson");
document.body.appendChild(button);

Aplicar mutadores a un elemento DOM existente

const button = document.createElement("button");
mutate(button)(
 domAttribute({ innerText: "hey you!" });
);
document.body.appendChild(button);

(async) run(…callbacks) → {any}

Ejecuta una serie de funciones, propagando una entrada nula, la cuál se actualiza con el valor retornado por el callback y es pasado al siguiente. Si el callback es asíncrono o devuelve una promesa, este esperará a que termine antes de ejecutar el siguiente.
